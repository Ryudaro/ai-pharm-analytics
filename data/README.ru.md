# Требования к данным и их структура

## Назначение данных

Система AI-Pharm Analytics работает с историческими данными о продажах
лекарственных препаратов и предназначена для построения прогнозов спроса
по каждому товару отдельно.

Прогнозирование выполняется на уровне конкретного товара без объединения
по категориям, группам или аптечным организациям.

Каждый прогноз строится независимо для одного товара.

---

## Источники данных

Система предполагает использование структурированных табличных данных
о продажах, полученных из учётных или отчётных систем аптек.

Типичные источники данных:
- локальные выгрузки из аптечных учётных систем,
- отчёты о продажах в формате CSV или Excel,
- агрегированные табличные отчёты по фактическим продажам.

Система не требует прямого подключения к базам данных или API и работает
с локально сохранёнными файлами.

---

## Гранулярность прогнозирования

Прогнозирование осуществляется на уровне **отдельного товара**.

Основные характеристики:
- каждый товар моделируется независимо,
- агрегирование между товарами не используется,
- иерархическое или групповое прогнозирование не предполагается.

Один временной ряд соответствует одному товару.

---

## Ожидаемая структура данных

Входной набор данных должен содержать следующие поля:

| Название поля     | Тип данных      | Обязательное | Описание |
|-------------------|------------------|--------------|----------|
| `date`            | Дата             | Да           | Дата наблюдения продаж |
| `product_id`      | Строка / идентификатор | Да     | Уникальный идентификатор товара |
| `sales_quantity`  | Числовой          | Да           | Количество проданных единиц товара за указанную дату |

Дополнительно могут присутствовать:
- идентификатор аптеки,
- наименование товара,
- вспомогательные атрибуты.

Наличие этих полей не является обязательным для работы прогнозных моделей.

---

## Требования к временным рядам

- временной индекс должен быть явно задан через поле `date`;
- наблюдения должны быть упорядочены по времени;
- используется фиксированный временной шаг (день, неделя или месяц);
- пропущенные даты должны быть явно представлены или обработаны на этапе предобработки.

Длина временного ряда по каждому товару должна быть достаточной
для обучения и оценки моделей прогнозирования.

---

## Допущения и ограничения

Система предполагает, что:
- данные отражают фактический реализованный спрос;
- значения в поле `sales_quantity` являются неотрицательными;
- возвраты, списания и корректировки либо исключены, либо учтены заранее.

Система **не моделирует**:
- влияние цены,
- промо-активности и маркетинговые кампании,
- взаимозаменяемость товаров.

Указанные факторы находятся за пределами текущей области применения системы.

---

## Политика работы с данными

Реальные данные о продажах, коммерчески чувствительная информация и
конфиденциальные наборы данных намеренно не хранятся и не публикуются
в данном репозитории.

В репозитории могут размещаться:
- описания форматов данных,
- схемы и контракты данных,
- синтетические или демонстрационные примеры.

Все реальные данные остаются вне системы контроля версий.
